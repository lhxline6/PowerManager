Identify the Application for Replay Debugging
When you start a replay debugging session, the Integrated Virtual Debugger uses the current Visual Studio Startup Project in the solution to determine which executable file in the recording to debug. If your Visual Studio solution has more than one project that builds an executable file, you must set the project you want to debug as the StartUp Project before you replay.
To identify the application for replay debugging
1
In the Visual Studio Solution Explorer window, right-click the project that you want to replay debug.
2
Select Set As StartUp Project.
